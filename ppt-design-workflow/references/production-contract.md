# PPT 三阶段生产验收合同

仅在执行相应阶段时读取对应章节。本文件把用户原始三版提示词整理为可验证的生产约束。

## 目录

- 阶段 1：内容大纲与逐页内容
- 阶段 2：三版完整 PPT 拼图
- 阶段 3：高保真可编辑 PPTX
- 最终验收清单

## 阶段 1：内容大纲与逐页内容

### 输入

- 用户导入的源文件，或明确的 PPT 需求；
- 指定页数；若未指定，依据演示时长和内容量提出建议页数；
- 受众、场景、主题、语言、品牌限制与必须保留的信息。

### 输出结构

1. 演示目标与一句话主线。
2. 章节级大纲。
3. 逐页内容表，每页至少包含：
   - 页码和页面类型；
   - 页面标题；
   - 本页要回答的问题；
   - 正文要点、关键数字或原文引用位置；
   - 数据/事实来源或“待提供”标记；
   - 推荐视觉形式；
   - 与前后页的叙事关系。
4. 风险、缺口与待确认项。

### 约束

- 达到指定页数，但不以重复内容凑页。
- 关键结论必须能追溯到源材料或明确标记为建议/假设。
- 不新增未经确认的数据、品牌、Logo、人物、产品、案例或来源。
- 用户确认后冻结页数、页序、故事线和逐页核心内容，除非用户明确要求修改。

## 阶段 2：三版完整 PPT 拼图

### 任务目标

本阶段只探索视觉方向。通过三张全套 PPT 拼图比较整体风格、页面节奏、内容适配度和高级感。

### 输出要求

1. 生成三张独立拼图：方案 A、方案 B、方案 C。
2. 每张拼图必须包含整套 PPT 的所有页面缩略图，并保持正确页序。
3. 每个缩略页均为 16:9 横版页面；拼图采用整齐网格并清晰标注页码。
4. 三版风格必须明显不同，不得只更换颜色。
5. 每版内部必须使用统一视觉系统：
   - 字体与字号层级；
   - 色彩系统；
   - 背景语言；
   - 图表与数据可视化；
   - 图标与插画；
   - 卡片/模块；
   - 页脚和页码。
6. 每版均应像完整、正式、可实现的商业 PPT，不得像零散草稿或模板拼贴。
7. 拼图文字可适当缩小，但主标题、关键数字、核心图表和页面结构必须可辨识。

### 内容约束

- 严格使用已确认的大纲和逐页内容。
- 不重写故事线，不删页、不加页、不改页序。
- 不新增未经确认的数据、品牌、Logo、人物、产品或来源。
- 图像生成模型无法可靠呈现长文本时，优先保证布局、标题位置、数字层级和图表结构；不得以乱码代替需要用户判断的核心信息。

### 三版差异维度

三版至少在以下五项存在实质差异：

- 版式气质；
- 信息密度；
- 图表语言；
- 图片/插画使用方式；
- 背景处理；
- 视觉重心；
- 标题排版；
- 空间与留白节奏。

具体方向由内容决定。例如可从“极简编辑部报告”“深色电影感叙事”“模块化数据系统”“温暖人文纪实”“高端品牌画册”等方向中选择适配的三类，不固定套用。

### 视觉底线

- 专业、高级、清晰，避免默认 PPT 模板感。
- 图标保持同一风格；图表清楚、可理解、具备商业报告质感。
- 图片使用高质量真实感素材或高级商业插图，并与主题相关。
- 不伪造品牌 Logo、人物肖像、产品 UI 或未经确认的视觉资产。
- 商业、金融、研究或战略汇报优先可信度和清晰度，不追求夸张效果。

### 阶段边界

- 只输出方案 A、B、C 三张完整拼图和简短标签。
- 不生成 PPTX。
- 不生成逐页高清单图。
- 等待用户选定方向后再进入阶段 3。

## 阶段 3：高保真可编辑 PPTX

### 核心目标

采用“复杂视觉保真 + 主要文字可编辑”的混合还原策略。最大限度保留选中方案的整体设计质感、构图、层次与高级感，同时保证主要内容可在 PowerPoint 中直接编辑。

### 硬性要求

1. 不得把整页简单铺成一张背景图。
2. 主要文字不得整体作为图片保留，应使用 PPT 原生文本框重建。
3. 必须尽量可编辑的文字包括：
   - 主标题和副标题；
   - 正文段落和金句；
   - 关键数字、关键词、单位和结论；
   - 标签、注释、章节名、页脚和页码；
   - 简单图表中的核心标签。
4. 复杂视觉区域内部的小字若拆分会明显破坏质感，可以保留为图片，但必须在最终说明中标注。
5. 不得为了全量可编辑，把复杂视觉重建成低质感的默认图形、默认图表或模板化卡片。

### 分层还原策略

1. 识别页面结构，划分为“复杂视觉资产层”和“可编辑信息层”。
2. 不直接放大拼图缩略页。根据选定方向逐页生成、提取或重建高清视觉资产。
3. 复杂视觉资产层可使用高清图片：
   - 照片、插画、3D 图和拼贴；
   - 复杂图表、流程图、循环图和信息图；
   - 复杂背景、材质、纹理、光影、景深；
   - 玻璃拟态、金属质感、柔光和高成本投影；
   - 品牌 Logo、商标、产品 UI、人物、商品图和官方图标的合法原始素材。
4. 可编辑信息层优先使用原生对象：
   - 文本框；
   - 简单线条和分隔线；
   - 基础几何框、标签、按钮和色块；
   - 页码、基础表格和简单图表。
5. 需要替换图片内文字时，优先生成无字版本；否则以背景色、渐变近似或局部遮罩覆盖，再叠加原生文字，避免重影。
6. 对无法准确反推数据的图表，只按视觉比例近似重建，并标记“数据为视觉近似”。

### 文字重建

- 匹配原方案的字体气质、字号层级、字重、颜色、行距、字距、对齐与位置。
- 字体不可用时选择最接近的系统字体，并在最终说明中记录替代关系。
- 中文可优先考虑黑体、苹方、思源黑体或微软雅黑气质；英文和数字选择匹配的无衬线或衬线字体。
- 保留高亮、颜色变化、字重变化、编号、引号、单位、标点和刻意换行。
- 禁止错字、漏字、换行错误、文本溢出、重叠、截断和明显位置漂移。
- 若完全照搬导致原生文字观感明显变差，可在不改变含义和层级的前提下轻微微调。

### 设计还原

- 保持版式比例、视觉重心、阅读顺序和留白节奏。
- 保持颜色、对比度、明暗关系、光源方向、阴影强度和空间层次。
- 不以默认 PPT 图标、默认图表、默认阴影或模板卡片替代复杂视觉。
- 尽量匹配元素位置、大小、层级、裁切、圆角、描边、阴影、透明度和对齐关系。
- 对图表、流程图、矩阵、时间线、对比图、循环图和信息图，优先保持视觉关系与阅读逻辑。
- 当可编辑性会显著降低复杂元素质感时，复杂元素优先作为图片保留，但主要文字除外。

### 渲染与迭代

1. 输出 PPTX 后渲染全套页面预览。
2. 同时检查拼图总览和逐页预览，分别发现跨页节奏问题与局部版式问题。
3. 对照选中方案检查质感、比例、间距、颜色、层级、裁切和整体观感。
4. 对照确认内容检查页序、文字、数字、来源和图表含义。
5. 若差距明显，继续迭代并重新渲染，直到通过质量门。

## 最终验收清单

- [ ] PPTX 能正常打开，页面比例为 16:9。
- [ ] 页数、页序和内容与确认稿一致。
- [ ] 所有页面成功渲染，无空白页、错位或缺图。
- [ ] 主要文字均可编辑，无重影、溢出或截断。
- [ ] 简单形状、表格和图表优先可编辑。
- [ ] 复杂视觉达到选定方案的整体质感。
- [ ] 品牌、人物、产品和数据未被伪造。
- [ ] 视觉近似数据已明确标注。
- [ ] 字体替代和保留在图片中的文字已说明。
- [ ] 最终说明列出原生对象、图片资产和例外项。
