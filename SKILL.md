# PPT Design Workflow

## 工作原则

- 按“内容确认 → 视觉确认 → PPTX 实现”推进，避免内容与视觉同时漂移。
- 将用户提供的事实、数据、页数、页序、品牌资产和措辞视为约束，不擅自改写或补造。
- 在需要执行某一阶段时，先读取 [references/production-contract.md](references/production-contract.md) 中对应阶段的完整验收合同。
- 优先保证叙事清晰、信息可信、设计统一、主要文字可编辑和最终文件可用。
- 保留中间产物，使每个阶段均可独立确认、回退和迭代。

## 判断起点

根据现有材料从最早未完成阶段开始：

1. 只有源文件或模糊需求：从阶段 1 开始。
2. 已有确认的大纲与逐页内容：从阶段 2 开始。
3. 已有确认内容且已选视觉方案：从阶段 3 开始。
4. 只要求修改现有 PPTX：保留已确认的内容与风格，仅执行相关实现和质检步骤。

若关键输入缺失但可以安全推断，先明确假设再继续。页数、受众、场景或品牌规范会实质改变结果时，提出一个简短问题。不要重复询问可从源文件得到的信息。

## 阶段 1：搭建内容框架

1. 读取全部源文件和用户要求，提取目标、受众、演示场景、页数、时长、语气、必须出现的事实与可用视觉资产。
2. 区分“已给定事实”“合理组织”“待确认信息”。对缺失数据使用明确占位符，不编造数字、来源、Logo、人物或产品信息。
3. 设计完整故事线，并输出：
   - 一句话叙事主线；
   - 章节结构与每章作用；
   - 逐页清单：页码、标题、页面目的、核心内容、证据/数据来源、建议视觉形式；
   - 需要用户确认的少量关键问题。
4. 校验总页数、页序、信息密度、前后逻辑和结论支撑。
5. 停在内容确认门。用户明确确认前，不生成视觉拼图。

## 阶段 2：探索三版视觉方向

1. 读取阶段 2 验收合同，并锁定已确认的大纲、页数和页序。
2. 根据主题、行业、受众、展示环境和内容密度选择三种真正不同且适配的视觉系统。差异须覆盖版式气质、信息密度、图表语言、图片策略、背景处理、视觉重心与标题排版，而非只换配色。
3. 调用图像生成能力，为 A、B、C 各生成一张完整拼图。每张拼图包含全套页面缩略图，全部为 16:9 且按正确页序排列。
4. 只输出三张拼图和极简风格标签；此阶段不生成 PPTX，不生成逐页单图，不重写内容。
5. 检查三版是否统一、专业、清晰、可落地，且主要标题、关键数字、核心图表和页面结构可辨识。
6. 停在视觉确认门，等待用户选择 A、B、C 或提出组合修改意见。

如果用户明确要求全流程不中断，并授权代选视觉方向，可以选出最适合的一版继续；否则不得替用户越过视觉确认门。

## 阶段 3：还原并制作 PPTX

1. 读取阶段 3 验收合同，将选中拼图作为视觉系统参考，而不是直接使用低分辨率缩略图。
2. 为每页拆分两层：
   - 复杂视觉资产层：照片、插画、3D、纹理、复杂信息图、复杂背景和高成本视觉效果；
   - 可编辑信息层：标题、正文、数字、标签、注释、页脚、基础形状、简单图表和表格。
3. 为复杂视觉区域生成或提取足够分辨率的素材；优先生成不含正文文字的视觉资产，再以原生文本覆盖。不得把整页简单铺成一张背景图。
4. 使用 PPT 原生文本框、形状、线条、表格和简单图表重建主要信息，匹配选定方案的字号层级、字重、颜色、行距、对齐、位置和留白。
5. 对图片中原有文字使用局部遮罩或无字重建后再叠加可编辑文本，避免重影。品牌 Logo、商标、人物、产品 UI 和官方图标不得凭空仿造。
6. 无法准确反推的图表数据，只可按视觉比例近似并明确标注“数据为视觉近似”。
7. 生成 PPTX 后渲染所有页面预览，逐页对照内容合同和选定视觉方案，迭代修复明显差异。

## 质量门

交付前至少完成以下检查：

- 内容：页数、页序、标题、数字、单位、来源和结论均与确认稿一致。
- 文字：无错字、漏字、异常换行、溢出、重叠、截断或位置漂移。
- 视觉：全套使用统一字体层级、色彩、图表、图标、卡片、页脚与页码系统。
- 版式：保持 16:9、阅读顺序、视觉重心、对齐、留白和跨页节奏。
- 可编辑性：主要文字为原生文本；简单元素优先原生重建；复杂视觉保留质感。
- 文件：PPTX 可打开，全部页面成功渲染，未出现字体替换导致的明显错版。

## 交付物

最终提供：

1. 可编辑的 `.pptx` 文件；
2. 全套渲染预览图或预览 PDF；
3. 简短实现说明，分别列出：可编辑文本/形状/图表、图片资产、仍嵌在复杂视觉区域中的文字、视觉近似数据及字体替代。

不要额外提交无关过程文件。若用户只要求阶段 1 或阶段 2，只交付该阶段产物。
